# 魔方简历 (Magic Resume) - 产品需求说明书

> 版本: v2.0.5 | 最后更新: 2025-05-01

---

## 一、产品背景与目标

### 1.1 市场痛点

当前市面上的简历工具普遍存在以下问题：

1. **隐私风险**：用户简历数据上传至云端，存在泄露风险
2. **收费门槛**：高级模板、PDF 导出等核心功能需付费
3. **不够灵活**：模板固定，难以深度自定义排版
4. **缺乏智能**：无 AI 辅助，纯手动编写效率低
5. **格式兼容差**：导出 PDF 样式错乱，投递效果差

### 1.2 产品目标

打造一款**开源免费、隐私优先、AI 驱动、专业美观**的简历编辑器：

- ✅ 数据完全本地存储，不上传云端
- ✅ 所有核心功能免费开源
- ✅ AI 辅助润色、语法检查、PDF 识别
- ✅ 8 套专业模板 + 深度自定义
- ✅ 高精度 PDF 导出，100% 还原排版

### 1.3 成功指标

| 指标 | 目标值 |
|------|--------|
| GitHub Stars | > 1000 |
| 月活跃用户 | > 5000 |
| PDF 导出成功率 | > 95% |
| 用户满意度 | > 4.5/5 |

## 二、用户画像

### 2.1 主要用户群体

#### 用户 A：应届毕业生小张
- **背景**：计算机专业，正在找第一份工作
- **痛点**：没有写过正式简历，不知道该怎么写
- **需求**：有示例数据参考，AI 帮助润色表达
- **使用场景**：选择模板 → 修改示例数据 → AI 润色 → 导出 PDF

#### 用户 B：资深工程师老李
- **背景**：5 年工作经验，经常更新简历
- **痛点**：需要维护多份针对不同岗位的简历版本
- **需求**：多简历管理、快速复制修改、本地备份
- **使用场景**：复制已有简历 → 针对性修改 → 本地文件夹同步

#### 用户 C：隐私敏感用户小王
- **背景**：安全行业从业者
- **痛点**：不信任任何云端简历服务
- **需求**：完全离线可用，数据不出本机
- **使用场景**：断网编辑 → 本地文件夹备份 → 仅在导出时联网

## 四、交互设计规范

### 4.1 设计原则

1. **即时反馈**：所有操作在 300ms 内给出视觉反馈
2. **防错设计**：危险操作（删除）需二次确认
3. **渐进披露**：高级功能默认隐藏，按需展开
4. **一致性**：相同功能在不同位置表现一致

### 4.2 动画规范

| 场景 | 动画效果 | 时长 |
|------|----------|------|
| 页面入场 | fade + slide | 300ms |
| 模态框打开 | scale + fade | 200ms |
| 面板收起/展开 | width transition | 300ms |
| 按钮悬停 | scale 1.05 | 150ms |
| 拖拽排序 | lift + ghost | 实时 |
| Toast 提示 | slide up + fade | 300ms |

### 4.3 错误处理

| 场景 | 处理方式 |
|------|----------|
| 导出失败 | Toast 错误提示，保留重试按钮 |
| AI 调用失败 | Toast 错误提示，建议检查配置 |
| 导入格式错误 | 明确提示"配置文件格式错误" |
| 浏览器不支持 FSAA | 提示使用现代浏览器 |
| 内容超出一页 | 提示精简内容或关闭一页纸模式 |
